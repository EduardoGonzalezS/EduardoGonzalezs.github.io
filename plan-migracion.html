3. Fases de Migración
Roadmap en 4 fases incrementales de 16 semanas en total
1
Fase 1 — Setup e Infraestructura
Semanas 1–2
2 semanas
  • Crear solución .NET 9 con proyecto Razor Pages
  • Escribir Dockerfile multi-stage (build + runtime)
  • Configurar Artifact Registry en GCP
  • Desplegar servicio Cloud Run vacío (health check)
  • Pipeline CI/CD: Cloud Build trigger en push a main
  • Configurar Secret Manager para connection strings
Entregable: URL de Cloud Run funcional que responde 200 OK con página index placeholder
2
Fase 2 — Core Pages y Autenticación
Semanas 3–6
4 semanas
  • Crear _Layout.cshtml con sidebar + topbar (HTML existente)
  • Integrar ASP.NET Identity: login, logout, roles (Admin/Gerente/Cajero/Farmacéutico)
  • Migrar Dashboard → Razor Page con datos simulados en C#
  • Migrar Ventas → Razor Page + filtros server-side
  • Migrar Inventario → Razor Page + paginación
  • Proteger rutas con [Authorize(Roles="...")]
Entregable: Login real con roles funcional + 3 módulos principales navegables en Cloud Run
3
Fase 3 — Capa de Datos Real
Semanas 7–12
6 semanas
  • Modelar entidades EF Core: Producto, Sucursal, Usuario, Venta, OrdenCompra, Inventario, Transferencia, Proveedor, Alerta
  • Crear instancia Cloud SQL (PostgreSQL 16) + configurar VPC
  • Ejecutar migraciones EF Core (dotnet ef database update)
  • Seed data: 8 sucursales, usuarios por rol, productos de muestra
  • Conectar todos los módulos (Transferencias, Usuarios, Proveedores, Compras, Alertas) a BD real
  • Configurar Secret Manager para connection string de producción
Entregable: Base de datos real en Cloud SQL — todos los módulos leen y escriben datos persistentes
4
Fase 4 — Paridad Completa y Go-Live
Semanas 13–16
4 semanas
  • Sistema de permisos granulares por módulo y sucursal
  • POS con AJAX (añadir al carrito sin recarga de página)
  • Alta masiva CSV: FileReader → validación server-side → EF Core bulk insert
  • QR Scanner integrado (html5-qrcode → API endpoint)
  • Auditoría de acciones (log de cambios en BD)
  • Pruebas de carga + ajuste de recursos en Cloud Run
Entregable: Sistema en producción con paridad completa — Go-live oficial
4. MCP Tools para la Migración
Integraciones de Claude Code que aceleran cada etapa del proceso de migración
Filesystem MCP Activo ahora
Ya configurado en este proyecto. Claude Code lee el HTML de C:\FM\ y genera directamente los Razor Pages equivalentes (.cshtml), modelos C# y migrations de EF Core. Sin copiar y pegar manualmente.
Lee archivos HTML actuales Escribe archivos .cshtml/.cs Sin configuración adicional
GitHub MCP Requiere PAT
Con un Personal Access Token de GitHub, Claude Code puede crear branches por fase (fase/1-setup, fase/2-core-pages...), hacer commits, abrir Pull Requests y revisar código. Control de versiones completo desde el chat.
Branches por fase Commits automáticos Pull Requests desde Claude
Google Cloud MCP (gcp-mcp) Requiere instalación
Una vez instalado (npx @google-cloud/mcp + autenticación gcloud), Claude Code puede desplegar a Cloud Run, consultar logs en tiempo real, administrar Cloud SQL, configurar variables de entorno en Secret Manager y revisar métricas — todo desde el chat sin salir de Claude Code.
Deploy Cloud Run Logs en tiempo real Cloud SQL + Secret Manager
Con los 3 MCPs configurados, toda la migración puede ejecutarse desde Claude Code: leer el HTML actual → generar Razor Pages → commitear a GitHub → hacer push de imagen Docker → desplegar en Cloud Run → verificar logs. Un ciclo completo sin abrir terminales ni consolas externas.
